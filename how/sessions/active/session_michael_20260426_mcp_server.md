## Design decisions (CTO-delegated)

- **Stdio default; no remote MCP in v1.** Local stdio matches host-blind posture: keys never leave the machine. Remote SSE would require server-side seed handling, which violates ADR-003. Hosted MCP can ship later as a thin courtesy wrapper that *only* exposes view/unfurl (no key-bearing operations).
- **Open seed mode is default** (matches web `feedback_zero_friction_default`). Passphrase opt-in via `SENDWYRD_PASSPHRASE` env var or `--passphrase-cmd "pass sendwyrd"` for `pass` / `gpg` integration.
- **Verbs only, no subscriptions.** `sendwyrd_inbox` is a one-shot read, not a polling/streaming surface. No `_watch`, no `_subscribe`, no auto-reply (per `feedback_anti_scope_creep_relay_layer` and ADR-010).
- **Config dir:** `${XDG_CONFIG_HOME:-~/.config}/sendwyrd/` with `config.json`, `seed.{enc,open}.json`, `history.json`. Same encrypted-seed format as the web app (v2 record from `@sendwyrd/core/seedStore`) so a future "import web seed → MCP" UX is trivial.
- **Origin override:** `SENDWYRD_ORIGIN` env var (default `https://sendwyrd.com`) so dev/staging targets work.
- **No `npm publish` in this PR.** Workspace install only. Publishing to npm under `@sendwyrd/mcp` requires `npm login` against the `@sendwyrd` org which the user has flagged as a separate task in the rolling backlog.

## Design decisions banked

- Stdio default; no remote MCP in v1 (would require server-side seed handling — violates ADR-003).
- Open seed mode is default (matches web zero-friction default). Passphrase opt-in.
- Verbs only — no subscriptions, polling, watch, or auto-reply tools (per `feedback_anti_scope_creep_relay_layer` and ADR-010).
- Bundled distribution (esbuild) so the MCP runs as plain Node without workspace-aware module resolution. Consumers can install via npm without needing tsx, ts-node, or the workspace.
- Same on-disk seed format as the web app, so a future "import web seed → MCP" UX is trivial (the encrypted record is bytewise compatible).
